摘要

1.前言

1.1 研究背景與動機

近年來全球資訊安全事件不斷發生，
資訊犯罪案例及手法更不斷翻新，
為保護企業資訊相關資產維持營運，企業高層主管負有實施資訊安
全之責任，而國際認定的資訊安全管理系統（Information Security
Management System 簡稱ISMS）便是一套有系統又能有效控制之管理方
法，採用的正是 ISO27001 資訊安全管理系統所要求的管理標準。

2.背景知識與相關研究

2.1.ISMS

ISMS (Information Security Management System)
是一套有系統地分析和管理資訊安全風險的方法。

標準文件 ISO27001 說套系統利用 PDCA 流程來持續改善機關之資訊安全管理制度。

PDCA 流程包含以下四個階段：

規劃 建立與管理風險及改進資訊安全相關之ISMS的政策、目標、過程及程序，以產生
     與組織整體政策和目標相一致的結果。

執行 實做與運作ISMS的政策、控制措施、過程和程序。

檢查 依據ISMS政策、目標及實際經驗、評鑑及在試用時測量過程績效，
     並將結果回報給管理階層審查。

行動 基於ISMS內部稽核與管理階層稽查結果或其它相關資訊採取矯正與預防措施，
     以達成ISMS的持續改進。

而標準文件 ISO27002 描述各種控制措施，並由機關參考實作，
以達成消除或減少資安風險發生機率。

2.2.openNMS

openNMS 是由許多個人及組織，所共同打造的。
從最初到2002 年，程式碼由Oculan Corporation 開發並以GPL 釋出，
後來專案的管理移交給Tarus Balog。
openNMS 是世界上第一個用開放原始碼模式開發的企業級網路管理系統。
openNMS 主要是用 Java 開發的，
雖然為了實作例如ICMP 這類東西而用到一些 JNI 來呼叫C 語言程式碼。
目前openNMS 支援Linux 的很多不同發行版本，Solaris 和Mac
OSX。
openNMS 目前提供了2 種版本，穩定版（1.2.9
版）及測試版（1.5.93 版）目前openNMS 的贊助企
業是The openNMS Group，它們也擁有openNMS
的商標。

openNMS 的操作介面是Web 介面，能夠幫助
IT 管理部門持續的監控網路設備的運作狀態，作業
系統如Windows、Linux、Unix 等，支援SNMP 協
定，只要監控物件支援SNMP 就可以被監控，提供
非常靈活的功能從而有利於管理，內建故障事件管
理以提供故障事件的記錄和分析，可以快速部署，
簡單的圖形介面令使用者迅速掌握，因此降低操作
的複雜度，提高管理效率，如圖4 所示。

2.2.1.主要功能

功能清單：
  1.自動偵察網路節點服務 
  2.事件管理
  3.資產管理
  4.效能管理

自動偵察或手動設定網路服務
  OpenNMS 可以自動搜尋本局提供之網路服務，也可以手動輸入網路服務。

事件及通知管理
  OpenNMS 可以主動產生自定義事件事件或從外部來源接收事件，
  外部源如SNMP陷阱、syslog等等。

  OpenNMS 可集中管理網路事件，其可每秒處理數千事件，
  並提供相關法來自動清除事件、轉換事件，合併重複的事件為一個警告。

  一旦辨識重要事件，OpenNMS 會發送通知。
  通知管道可為電子郵件、SMS 等等。

服務保證

  OpenNMS的是在開始的時候，服務水平協議（SLA）為重點的太多的管理工作。
  該應用程序帶有大量的服務監視執行綜合交易範圍從簡單的ICMP請求（ping）
  或端口檢查，透過複雜的網站監控和往返電子郵件測試。

  可以生成詳細的報告上提供的服務，這是非常容易定制投票率，
  以及配置計劃停機時間。

績效評估

有一個數字的數據採集器在OpenNMS的，包括支持SNMP和JMX協議。該收集器可以
檢索任何HTTP網頁，並使用正則表達式來提取值，因此擴展的應用程序整合
OpenNMS的可以簡單的寫一個文本文件，可以訪問 Web服務器。

所有數據存儲可以測試對閾值，不僅一般的“高”和“低”的觸發閾值，但基於一個
相對或絕對價值的變化。這些閾值可以產生事件進行管理的事件和通知系統。

性能數據可以繪製，和“TOP N”報告可以輕鬆地找出產生問題的領域內網。

這是一個非常高層次的概述OpenNMS的應用。得到它，安裝和探索，看看是多麼
強大，靈活和可擴展的應用程序即可。
