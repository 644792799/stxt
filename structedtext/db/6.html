6.應用為 DBMS 的 INDEX

*BT 可作為大量資料的 index 的實作 像是 Oracle DBMS 便是使用 B*T 作為 Index 的結構。

DBMS 使用 INDEX 的目的如下： *加快搜尋速度

*實作 UNIQUE CONSTRAINT 由於 DBMS 常需要某欄位的值不含有重覆值，像是 RDBMS 中 table 的主鍵，故要能快速判斷新插入的值是否重覆，便要使 為主鍵建立 INDEX。 B樹索引是數據庫中存取和查找文件(稱為記錄或鍵值)的一 種方法。B-tree算法減少定位記錄時所經歷的中間過程，從而加快存取速度。
